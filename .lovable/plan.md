

## Fix QR Code Download — Validation Always Fails

### Problem

The QR download button calls `validateQrFromImageData(imageData, url)` without passing a decoder. The default decoder is `() => null`, so validation always returns `scan_failed` and blocks every download.

### Solution

Use the browser's built-in `BarcodeDetector` API (available in Chrome/Edge) as the QR scanner. If `BarcodeDetector` is not available (Firefox/Safari), skip validation and allow the download — the QR was just generated by `qrcode.react` so it's trustworthy by construction.

### Changes

| File | Change |
|------|--------|
| `src/components/QRCodeDialog.tsx` | Replace the synchronous `validateQrFromImageData` call with an async flow: try `BarcodeDetector` to decode the canvas, validate URL match, fall back to allowing download if `BarcodeDetector` is unavailable |

### Technical details

Replace lines 212-225 in `QRCodeDialog.tsx` with:

```typescript
// Validate QR code before download using BarcodeDetector if available
const performDownload = () => {
  const link = document.createElement('a');
  link.download = `${productName.replace(/[^a-z0-9]/gi, '_')}_qr.png`;
  link.href = canvas.toDataURL('image/png');
  link.click();
};

if ('BarcodeDetector' in window) {
  const detector = new (window as any).BarcodeDetector({ formats: ['qr_code'] });
  const blob = await new Promise<Blob | null>(r => canvas.toBlob(r));
  if (blob) {
    const bitmap = await createImageBitmap(blob);
    const results = await detector.detect(bitmap);
    if (results.length > 0 && results[0].rawValue === url) {
      performDownload();
    } else if (results.length === 0) {
      console.warn('BarcodeDetector could not read QR — allowing download (QR was just generated)');
      performDownload();
    } else {
      toast.error(t('qrDialog.urlMismatch', 'QR code validation failed - URL does not match'));
      return;
    }
  } else {
    performDownload();
  }
} else {
  // No BarcodeDetector available — trust the generated QR
  performDownload();
}
```

The `img.onload` callback will need to become `async`. The `validateQrFromImageData` import can be removed from this file (it remains available for tests).
